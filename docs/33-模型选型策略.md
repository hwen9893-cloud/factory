# 1. center.model 多后端封装原则

`center.model` 对上层（skill/factory）暴露**统一 OpenAI 兼容接口**，底层后端可切换，呼应架构原则 ⑥ Replaceable。

```
skill → center.model（统一接口）→ ┌ 本地 Ollama / MLX（Llama3 / Qwen ...）
                                  ├ 云 LLM API（按需）
                                  └ 专用模型（TTS / 视觉 / 视频）
```

> 选型不是"一次定终身"，而是**为每类任务选最优后端，并保留切换能力**。

# 2. 基座候选对比（中文爽文写作）

> 用户已指定 **Llama3** 为基座。但需正视：Llama3 系列的**中文创作能力弱于主流中文开源模型**。下表为选型依据，最终以 §3 盲测结论为准。

| 模型 | 中文爽文能力 | Mac 本地可行性（量化） | 备注 |
|------|------------|----------------------|------|
| **Llama3 / 3.1 (8B/70B)** | 中（中文偏弱，需提示工程/微调） | 70B-4bit 可跑（≥128GB） | 用户指定；英文强、中文需增强 |
| **Qwen2.5 (72B)** | 强（中文原生强） | 72B-4bit 可跑 | 中文爽文首选候选 |
| **DeepSeek-V2/V3** | 强 | 体量大，部分需云 | 推理/长文强 |
| **GLM-4 (9B/视情)** | 较强 | 9B 本地轻松 | 轻量本地选项 |
| **Yi-1.5 (34B)** | 较强 | 34B 本地流畅 | 折中选择 |

# 3. Llama3 的定位决策

采用**实测后决策**，而非拍脑袋：

1. **盲测**：用同一修仙故事种子，让 Llama3-70B 与 Qwen2.5-72B 各生成同段落，由剧情设计盲评（用 `41` 角色量化 + `72` 内容评测打分）。
2. **判定**：
   - 若 Llama3 经提示工程后**达到可用**（人工修改率 ≤ 20%，见 `22` 标准 #2）→ 保留 Llama3 为**默认写作后端**。
   - 若中文质量系统性落后 → 将 Llama3 降为**可选后端**，默认改用 Qwen2.5/DeepSeek，Llama3 保留用于英文/特定任务。
3. **中文增强（若坚持 Llama3）**：
   - 优先**提示工程 + few-shot 设定注入**（零训练成本）。
   - 次选社区中文增强权重（Llama3-Chinese 类）。
   - 继续预训/SFT 成本高，Phase 1 不做。

> 结论写回本文档 §3 并冻结；后续若换模型，只改 `center.model` 配置，不动上层 skill。

# 4. 分任务选模型

| 任务 | 推荐后端 | 部署 |
|------|---------|------|
| 小说写作 | Llama3-70B 或 Qwen2.5-72B（按 §3 实测） | 本地 |
| 设定审校 / QA | 同写作模型或更小的快速模型 | 本地 |
| Embedding（检索/一致性） | bge / gte 系列中文 embedding | 本地 |
| TTS / 配音 | 专用 TTS（多说话人），每 Character 绑定固定 speaker | 本地为主 |
| 图像生成 | Flux / SD 系 + 角色 LoRA | 本地试色 / 云放量 |
| 视频生成 | reference 条件的视频模型 | 云 GPU |

# 5. 量化格式

| 格式 | 适用栈 | 说明 |
|------|--------|------|
| **GGUF** | Ollama / llama.cpp | 通用、量化档位丰富（Q4_K_M 等） |
| **MLX** | Apple MLX | Apple Silicon 原生最优 |

> 默认 4-bit（Q4_K_M 或 MLX 4bit）平衡质量与内存；审校等对质量敏感的任务可上 5/8-bit。

---

# 6. 模型评测与回归

- 每次更换模型/量化档位，跑 `72-评测工程EvalHarness.md` 的内容评测套件，与基线对比。
- 写作质量回归项：设定冲突率、人工修改率、角色量化总分分布。
- 防止"换模型导致中文质量悄悄退化"——纳入回归监控。
