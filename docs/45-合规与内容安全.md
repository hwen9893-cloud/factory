# 1. 文档目的

AI 生成内容在中国平台（抖音/B站/番茄/喜马拉雅）发布面临**双重合规要求**：
① 法规层：《生成式人工智能服务管理暂行办法》（2023）、《网络信息内容生态治理规定》
② 平台层：各平台独立的内容审核红线与 AI 标识规范

本文档把合规嵌入流水线，而非"上线前一次性通读政策"。

# 2. IP 与改编授权（创作起点的合规）

## 2.1 内容来源分类

| 来源 | 合规状态 | 处理 |
|------|---------|------|
| **自有 IP**（原创世界观/角色） | ✅ 无需授权 | 推荐首选；在 `42` 知识库注明"原创 IP" |
| **改编自公版/古典作品** | ✅ 作品版权已过期，但衍生需核查 | 标注来源；不照搬文句 |
| **改编自有版权网文** | ⚠️ 须取得改编授权 | **未获授权前不得发布**；联系版权方或换用原创 IP |
| **角色/设定参考已有 IP** | ⚠️ 相似度风险 | `center.qa` 增加原创性检查（见 §4.1） |

> **Phase 0 强烈建议使用自有 IP**——版权纠纷可以让整条流水线的成果归零。

## 2.2 自有 IP 登记

将每个 IP 的基本信息写入 `center.knowledge`，`knowledge.ip.*` 条目：

```json
{
  "ip_id": "knowledge.ip.0001",
  "title": "string（作品名）",
  "type": "original | adaptation",
  "source_ref": "string（若改编，填原著信息 + 授权文件链接）",
  "copyright_owner": "string",
  "registration": "string（著作权登记号，可空）"
}
```

# 3. AIGC 标识要求（中国法规）

## 3.1 法规要求摘要

《生成式人工智能服务管理暂行办法》第 12 条：
> 提供者应当按照《互联网信息服务深度合成管理规定》对图片、视频等生成内容**进行显著标识**。

各平台在此基础上有更严格的自定义规则（持续更新）。

## 3.2 标识规范（适配各平台）

| 平台 | 标识要求（2026 基准，需持续核查） |
|------|-------------------------------|
| 抖音 | 视频封面/片尾必须包含"AI 生成"标识；后台申报创作辅助工具 |
| B站 | 投稿时勾选"AI 辅助创作"标签；封面加水印 |
| 快手 | 视频描述注明 AI 生成；后台工具申报 |
| 喜马拉雅 | 专辑描述注明 AIGC；音频片头/片尾播报 |
| 番茄小说 | 章节首部注明 AI 辅助创作 |
| YouTube | 勾选"包含人工智能生成内容"；Shorts 单独申报 |

> **铁律：任何平台发布均须标识，不标识视为违规，轻则限流，重则封号。**

## 3.3 `skill.novel.export` 标识注入

`skill.novel.export` 在输出内容时自动注入标识文本：

```python
AIGC_DISCLAIMER = {
  "fanqie": "\n\n【本文由 AI 辅助创作，已经人工审核】",
  "txt":    "\n\n[AI-Assisted Creation, Human-Reviewed]",
  "epub":   "<p class='aigc-notice'>AI 辅助创作 · 人工审核</p>",
}
```

视频/图像资产的水印由 `factory.publishing` 的 `skill.publish.label` 负责注入（见 §5）。

# 4. 内容安全红线（写作/生成约束）

## 4.1 修仙爽文高风险内容类型

修仙题材在中国平台存在以下审核风险点：

| 风险类型 | 具体内容 | 平台处理 | 流水线对策 |
|---------|---------|---------|-----------|
| **封建迷信** | 真实宗教修炼、算命占卜、现实神明名 | 限流/下架 | `center.qa` 规则：禁用真实宗教/神明名；炼气期/修仙期等架空语境可用 |
| **血腥暴力** | 过度残忍肢解、血腥描写超限 | 下架 | `skill.qa.compliance` 血腥度评分 ≤ 3/5 |
| **价值观问题** | 主角行为"三观不正"无因果、以暴制暴无代价 | 限流 | 每卷有"因果闭环"（爽点后需有相应代价/成长） |
| **政治敏感** | 影射现实政治/历史事件 | 立即下架 | 完全架空世界观，严禁与真实地名/人名对应 |
| **色情低俗** | 超尺度性描写 | 封号 | `skill.qa.compliance` 内容分级强制 ≤ PG-13 |
| **未成年保护** | 主角/反派为未成年的暴力/感情描写 | 下架 | 知识库强制：主角/关键角色设定为成年（≥18岁） |

## 4.2 `skill.qa.compliance`（合规质检 skill）

```python
# center.qa 的合规检查 skill，在 factory.novel QA 阶段后运行

class SkillQaCompliance:
    name = "skill.qa.compliance"
    
    checks = [
        "real_religion_name",    # 禁真实宗教/神明名
        "violence_level",        # 血腥度 0–5 评级
        "political_sensitivity", # 现实影射检测
        "sexual_content",        # 性描写分级
        "minor_protection",      # 未成年保护
        "karma_closure",         # 因果闭环（每卷末）
    ]
    
    def run(self, state):
        # 先跑规则引擎（快，高召回）
        rule_results = self.run_rules(state["body"])
        # 规则未明确的用 LLM 分类（慢，高精度）
        llm_results = self.run_llm_classifier(state["body"])
        return merge_results(rule_results, llm_results)
```

**在 factory.novel 流水线中的位置**：
```
skill.qa.consistency（设定冲突）→ skill.qa.compliance（内容安全）→ export
```
两道 QA 都通过才允许 export，任一 `hard` 问题均需人工放行。

# 5. 发布前合规流程（`factory.publishing`）

## 5.1 更新后的 publishing 流水线

```
skill.publish.compliance_check   ← 终审：平台规则 + 标识核查
        │ 通过
        ▼
skill.publish.label              ← 注入 AIGC 水印/标识（图像/视频）
        │
        ▼
skill.publish.format             ← 平台格式适配
        │
        ▼
skill.publish.adapt              ← 多平台内容适配（标题/简介/话题标签）
        │
        ▼
skill.publish.push               ← 推送
        │
        ▼
skill.publish.report             ← 回传数据至 center.operation
```

## 5.2 `skill.publish.label` 规范

| 产物类型 | 标识方式 | 技术实现 |
|---------|---------|---------|
| 文本（小说/章节） | 首部/尾部文字声明 | `skill.novel.export` 已内置 |
| 图像（立绘/场景） | 角落水印 + EXIF 元数据 | PIL/EXIF 注入 `"AI Generated"` |
| 视频 | 片头/片尾标识帧 + 封面水印 | FFmpeg 叠层 |
| 音频（有声书） | 片头 TTS 声明 + 元数据标签 | ID3 tag |

# 6. 持续合规维护

| 触发 | 动作 | 负责人 |
|------|------|--------|
| 法规更新（每季度核查） | 更新 §3.2 平台标识规范表；若有 hard 变更更新 skill.qa.compliance 规则 | 代码 + 全员 |
| 平台政策变更（监控） | 对应平台 publishing 适配更新 | 代码 |
| 内容被下架/限流 | 复盘 §4 红线，更新规则集，重新审核历史内容 | 全员 |
| 新进入平台 | 补充该平台标识要求到 §3.2 | 代码 |

> **原则：合规是持续过程，不是上线前一次性检查。每次法规/平台规则变更都需要同步更新本文档和 `skill.qa.compliance`。**
