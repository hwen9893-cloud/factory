# 1. 为什么单独立项

原文档把"角色一致性差"放在风险表里，用一句"建立 Character ID"带过。这是低估。

**一致性不是众多风险之一，而是决定本项目能否成立的唯一关键技术难题。** 截至目前，长视频中的角色/场景一致性仍是行业普遍未解决的问题。如果这关过不了，第三阶段动画流水线、乃至整个 IP 化目标都是空中楼阁。

因此把它从风险表抽出，作为**时间盒预研子项目**先行验证，并设一道硬性关卡（M3.5）卡在图像资产（M3）与动画 MVP（M4）之间。

> 硬件叠加风险：本项目的视觉/视频生成依赖云 GPU（Mac 跑不动，见 `32` §3），故一致性预研需在云镜像中进行；落地分工见 `51-一致性Playbook.md`。

---

# 2. 一致性问题的四个维度

| 维度 | 含义 | 难度 |
|------|------|------|
| **身份一致性** | 同一角色每次生成长得/听起来一样 | 中 |
| **跨模态一致性** | 文字描写 ↔ 立绘 ↔ 视频 ↔ 配音 对齐 | 高 |
| **时序一致性** | ①单段视频帧间稳定 ②跨 50 章不漂移 | 极高 |
| **设定一致性** | 事实 / 时间线 / 关系不自相矛盾 | 中（偏工程） |

> 真正的难点是后两者：单张立绘一致相对容易，难的是"文→图→视频→声"四模态对齐，且在第 50 章仍不漂移。

# 3. 分维度技术路线（候选方案）

> 候选手段，非最终选型；预研结论写入 `51-一致性Playbook.md`。各手段的 Mac/云执行位置见 `51` §2。

## 3.1 身份一致性（图像）
- 每角色一套 **LoRA** + 固定 **canonical reference sheet**（多角度定妆图）
- **Reference-based 生成**（IP-Adapter / InstantID 类）做身份锚定
- 固定 seed 区间 + ControlNet 控姿态构图
- 兜底：生成后做 face restoration / 局部重绘修正

## 3.2 跨模态对齐（文 → 图）
- 角色"外观规格"（appearance spec）结构化存入 `center.asset`，字段化：发色/瞳色/服饰/标志物/体型…
- Prompt 由规格**确定性注入**，而非每次让 LLM 重新口述描写（避免 LLM 措辞漂移引入视觉漂移）

## 3.3 时序一致性（视频）
- **关键帧锁定**：先定关键帧，再 reference-based 生成中间帧
- 末帧→首帧链式衔接长镜头
- 帧间稳定优先选带 reference 条件的视频模型
- 兜底：逐帧 face swap / restoration 作为后处理回路

## 3.4 配音一致性
- 每 Character ID 绑定固定 voice clone / TTS speaker embedding，全系列复用

## 3.5 设定一致性（由 `center.qa` 承担）
- QA Agent 将新章节 / 新分镜与 `center.knowledge` 既有事实做 diff
- 对人物属性、时间线、宗门关系做约束检查，冲突即拦截并回报（规则集见 `42` §4）

# 4. M3.5 一致性验证关卡（验收门）

**位置**：M3（图像资产，第 6 月）与 M4（动画 MVP，第 9 月）之间。
**性质**：硬性 Go / No-Go 门，未通过不得进入动画流水线开发。

## 验收指标（启动基线，预研期校准）

| 维度 | 测试方法 | 通过基线（待校准） |
|------|----------|-------------------|
| 身份一致（图像） | 主角独立生成 N=20 张，与参考图做人脸 embedding 余弦相似度（如 ArcFace） | 任一张 ≥ 0.6；人工"是否同一角色"判定一致率 ≥ 90% |
| 跨模态（文→图） | 给定章节角色描写，生成图在 K 个外观属性上比对规格 | 属性命中率 ≥ 85% |
| 时序（视频） | 产出 1 段 10 秒片段 | 帧间身份漂移低于阈值，无灾难性形变；人工通过率 ≥ 80% |
| 设定（QA） | 向文本注入一组人工标注的设定冲突测试集 | QA Agent 冲突召回率 ≥ 80% |

> 数字标注为"待校准"；预研第一周先建测试集与度量脚本（归口 `72-评测工程EvalHarness.md`），用基线数据反推合理阈值，避免假精确。

## 不通过的处置
- 单维度未达标 → 在该维度迭代方案或更换技术路线，**不放行**
- 多维度系统性未达标 → 触发 de-scope 决策（例如先做半动态分镜/动态漫，而非全视频），重排 M4 范围（见 `51` §6）

# 5. 预研排期（时间盒）

| 周 | 任务 | 产物 |
|----|------|------|
| W1 | 建一致性测试集 + 度量脚本（embedding 相似度、属性命中、冲突召回） | 可复跑的评测基线 |
| W2–W3 | 图像身份 + 跨模态方案验证（LoRA / reference / 规格注入） | 图像维度结论 |
| W4–W5 | 视频时序方案验证（关键帧锁定 / 链式生成 / 后处理兜底） | 视频维度结论 |
| W6 | `center.qa` 设定冲突检查原型 + 整合 | 一致性 Playbook + Go/No-Go 建议 |

并行于 M2/M3 开展，不单独占用主线工期；其结论直接喂给 M3.5 关卡。
